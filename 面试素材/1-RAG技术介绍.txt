RAG（Retrieval-Augmented Generation，检索增强生成）技术详解

一、什么是 RAG

RAG 是一种将信息检索与大语言模型生成相结合的技术架构。它的核心思想是：在让大模型回答问题之前，先从外部知识库中检索最相关的信息片段，将这些片段作为"参考资料"注入到提示词中，再让大模型基于这些资料生成回答。

这样做的好处：
1. 减少幻觉：大模型不再凭空编造，而是基于真实文档回答
2. 知识可更新：不需要重新训练模型，只需更新知识库
3. 来源可追溯：每条回答都能追溯到原始文档
4. 领域可定制：可以针对特定领域的私有知识构建专属问答系统

二、RAG 的核心流程

RAG 系统分为两个阶段：离线建库阶段和在线查询阶段。

离线建库阶段：
第一步：文档加载。支持 PDF、TXT、Markdown 等多种格式的文档。
第二步：文本切分。将长文档切分成适当大小的文本块（chunk），通常 300-500 字为一个块，块与块之间保留 50 字左右的重叠，防止关键信息被切断。
第三步：向量化。使用 Embedding 模型将每个文本块转换成高维向量（通常在数百到数千维之间）。语义相似的文本在向量空间中距离更近。
第四步：向量存储。将向量和对应的原始文本存入向量数据库，如 ChromaDB、FAISS、Milvus 等。

在线查询阶段：
第一步：用户输入问题。
第二步：将问题同样进行向量化。
第三步：在向量数据库中检索与问题向量最相似的 Top-K 个文本块（使用余弦相似度或欧氏距离）。
第四步：将检索到的文本块作为上下文，与用户问题一起组装成 Prompt。
第五步：调用大语言模型生成回答。

三、核心技术细节

1. 文本切分策略
递归字符切分是最常用的方法。按照自然分隔符的优先级依次尝试：段落换行 > 句号 > 逗号 > 空格 > 字符级切分。这样可以尽可能保持语义的完整性。

2. Embedding 模型选择
常见的 Embedding 模型包括：
- OpenAI text-embedding-3-small/large：1536/3072 维，效果好但需付费
- BGE 系列：768/1024 维，中文效果好，开源免费
- Sentence-BERT 系列：384-768 维，轻量级

3. 向量数据库对比
- ChromaDB：Python 原生支持，轻量级，适合原型开发
- FAISS：Facebook 开源，C++ 实现，检索速度快
- Milvus：分布式向量数据库，适合生产环境
- Pinecone：云托管服务，免运维

4. 检索优化技巧
- 混合检索：结合关键词检索（BM25）和向量检索，取长补短
- 重排序：对初步检索结果用更强的模型重新排序
- 多路召回：用多种策略分别召回候选，再做融合

四、RAG 的典型应用场景

1. 企业知识库问答：员工可以基于公司内部文档提问，如规章制度、产品手册等
2. 客服系统：基于产品文档和历史工单自动回答用户问题
3. 法律文档分析：快速检索相关法条和判例
4. 学术研究：基于论文库回答学术问题
5. 个人知识管理：基于个人笔记回答问题

五、RAG 的进阶方向

- Agent 化：让 RAG 不只检索，还能主动调用工具、执行多步推理
- 多模态 RAG：支持图片、表格等非文本内容的检索
- Graph RAG：引入知识图谱，提升复杂关系的推理能力
- Self-RAG：让模型自己判断是否需要检索、检索结果是否相关
